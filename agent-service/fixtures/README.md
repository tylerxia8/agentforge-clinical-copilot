# W2 test fixtures — synthetic PDFs

Two synthetic PDFs for end-to-end testing of `/agent/extract`:

| File | Doc type | What's in it |
|---|---|---|
| `sample_lab_report.pdf` | `lab_pdf` | 8 lab results in a Quest-style layout: HDL, LDL, Total Cholesterol, Triglycerides, HbA1c, fasting glucose, creatinine, eGFR. Realistic values for an uncontrolled T2DM follow-up (4 H flags). Patient is the demo seed Farrah Rolle. |
| `sample_intake_form.pdf` | `intake_form` | New-patient intake with demographics, chief concern, 3 medications, 1 allergy (penicillin → rash), 3 family history entries. |

Both PDFs are generated by hand-rolled Node scripts — no npm
dependencies — so they regenerate identically on any machine. To
rebuild:

```bash
cd agent-service/fixtures
node make_sample_lab_pdf.js
node make_sample_intake_form.js
```

## Using them

**Via the embedded panel (full MVP path):**
1. Open Farrah's chart at the deployed OpenEMR
2. Click the paperclip in the chat panel
3. Pick the PDF you want to test
4. The filename hints (`lab` / `intake`) drive doc-type detection;
   override via the `confirm()` prompt for the other type
5. Wait ~10–30s; the panel renders the structured extraction
   summary plus a "source bbox match" line

**Via curl + a minted token (programmatic):**

```bash
# Mint an HMAC token (see agent-service/evals/run.py mint_token).
TOKEN="..."

curl -X POST https://copilot-agent-production-ba87.up.railway.app/agent/extract \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@sample_lab_report.pdf" \
  -F "doc_type=lab_pdf" \
  -F "document_reference_id=fixture-lab-001" | jq .
```

## Synthetic-data guarantee

PRD §"HIPAA-minded development" requires synthetic data only. Every
field on these PDFs (name, DOB, address, phone, email, accession
number, lab values, allergies, family history) is fabricated to
look plausible while not corresponding to any real person.
